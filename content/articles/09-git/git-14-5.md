# Git Plumbing vs Porcelain Commands

Git's commands are divided into two groups: **porcelain** (user-friendly high-level commands) and **plumbing** (low-level commands for building tools). Understanding this distinction is key to mastering Git's internals.

## The Analogy

Like plumbing in a house:
- **Porcelain** (toilet, sink) — What users interact with, polished and user-friendly
- **Plumbing** (pipes, valves) — The underlying infrastructure that makes it work

In Git:
- **Porcelain commands** (`git commit`, `git merge`, `git status`) — Designed for humans
- **Plumbing commands** (`git hash-object`, `git cat-file`, `git update-ref`) — Designed for scripts and tool builders

## Common Porcelain Commands

These are the commands you use daily:

```bash
git add          # Stage files
git commit       # Create a commit
git status       # View repository state
git log          # View history
git diff         # View changes
git branch       # Manage branches
git merge        # Merge branches
git push         # Upload to remote
git pull         # Download from remote
git clone        # Copy a repository
git checkout     # Switch branches/files
git rebase       # Rebase commits
git stash        # Stash changes
```

## Key Plumbing Commands

These operate directly on Git's object model:

### Object Manipulation

```bash
# Store a file as a blob object and return its hash
git hash-object -w filename.txt

# Read the content of any object
git cat-file -p <hash>    # Print content
git cat-file -t <hash>    # Print type (blob, tree, commit, tag)
git cat-file -s <hash>    # Print size

# Create a tree from the current staging area
git write-tree

# Create a commit object
git commit-tree <tree-sha> -p <parent-sha> -m "message"
```

### Reference Management

```bash
# Read a reference
git rev-parse HEAD
git rev-parse main

# Update a reference directly
git update-ref refs/heads/main abc1234

# Create a symbolic reference
git symbolic-ref HEAD refs/heads/new-branch
```

### Staging Area (Index)

```bash
# Read the staging area
git ls-files                    # List staged files
git ls-files --stage            # With object hashes
git ls-files --others           # Untracked files
git ls-files --ignored          # Ignored files

# Update the index manually
git update-index --add filename.txt
```

### Traversal and History

```bash
# Walk commit history from a starting point
git rev-list HEAD

# List objects reachable from commits
git rev-list --objects HEAD

# Find common ancestors
git merge-base main feature/login
```

### Network and Transfer

```bash
# Fetch from a remote manually
git fetch-pack <url>

# Send objects to a remote
git send-pack <url>
```

## Building Blocks: How Porcelain Uses Plumbing

`git commit` under the hood runs:
1. `git write-tree` — Creates a tree from the staging area
2. `git commit-tree` — Creates a commit object from the tree
3. `git update-ref` — Moves the branch pointer to the new commit

```bash
# Manually create a commit (what git commit does internally)
tree_sha=$(git write-tree)
parent_sha=$(git rev-parse HEAD)
commit_sha=$(echo "Commit message" | git commit-tree $tree_sha -p $parent_sha)
git update-ref refs/heads/main $commit_sha
```

## When to Use Plumbing Commands

- Writing Git-related scripts or automation
- Building Git frontends or tools (like IDEs)
- Understanding Git deeply for debugging
- Processing repository data programmatically

Plumbing commands have stable, machine-readable output — unlike porcelain commands which may change their output format between Git versions.
