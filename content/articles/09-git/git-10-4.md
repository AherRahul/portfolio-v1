# GitHub Issues — Tracking Work

GitHub Issues is a built-in issue tracking system that lets teams report bugs, request features, track tasks, and manage project work directly within a repository.

## What is an Issue?

A GitHub Issue is a discussion thread linked to a repository that represents:
- 🐛 A **bug** that needs to be fixed
- ✨ A **feature request** or enhancement
- 📋 A **task** or to-do item
- ❓ A **question** about the project
- 📝 **Documentation** improvements

## Creating an Issue

1. Go to the repository on GitHub
2. Click the **"Issues"** tab
3. Click **"New issue"**
4. Fill in:
   - **Title** — Short, descriptive summary
   - **Description** — Detailed information (steps to reproduce for bugs)
   - **Labels** — Categorize the issue (bug, enhancement, documentation)
   - **Assignees** — Who should work on it
   - **Milestone** — Which release/sprint this belongs to
   - **Projects** — Link to a GitHub project board

## Writing a Good Bug Report

```markdown
## Bug Description
The login form throws a 500 error when passwords contain special characters.

## Steps to Reproduce
1. Go to /login
2. Enter email: user@test.com
3. Enter password: P@ssw0rd!
4. Click "Login"
5. See error

## Expected Behavior
User should be successfully logged in.

## Actual Behavior
500 Internal Server Error is returned.

## Environment
- Browser: Chrome 120
- OS: macOS 14
- Version: v2.1.3

## Additional Context
Started after the v2.1.0 release. Passwords without special chars work fine.
```

## Labels

Labels categorize and filter issues:

| Label | Color | Description |
|-------|-------|-------------|
| `bug` | Red | Something is broken |
| `enhancement` | Blue | New feature or improvement |
| `documentation` | Light blue | Docs improvements |
| `question` | Pink | Need more information |
| `good first issue` | Light green | Good for newcomers |
| `help wanted` | Green | Extra attention needed |
| `wontfix` | White | Won't be addressed |

Create custom labels specific to your project.

## Linking Issues to Pull Requests

Reference an issue in a PR description or commit message to automatically close it when the PR merges:

```markdown
# In a PR description:
Closes #234
Fixes #234
Resolves #234
```

```bash
# In a commit message:
git commit -m "Fix password validation - closes #234"
```

When the PR merges to the default branch, GitHub automatically closes the linked issue.

## Issue Templates

Create issue templates to guide reporters (`.github/ISSUE_TEMPLATE/`):

```markdown
<!-- .github/ISSUE_TEMPLATE/bug_report.md -->
---
name: Bug Report
about: Create a report to help us improve
title: '[BUG] '
labels: bug
assignees: ''
---

## Describe the bug

## Steps to Reproduce

## Expected behavior

## Screenshots

## Additional context
```

## Milestones

Group issues into milestones (e.g., "v2.0 Release"):
- Track progress toward a goal
- Set a due date
- See completion percentage

## Issue Lifecycle

```
Open → In Progress → In Review → Closed
         (assign)       (PR)     (merged/resolved)
```

- **Open:** New, unresolved issue
- **Closed:** Resolved (fixed, declined, or duplicate)
- **Locked:** No more discussion allowed

## Filtering and Searching Issues

```
is:open label:bug                    # Open bugs
is:closed assignee:@me               # Closed issues assigned to you
label:enhancement sort:reactions     # Most-reacted enhancement requests
author:username                      # Issues created by someone
milestone:"v2.0"                     # Issues in a milestone
no:assignee is:open                  # Unassigned open issues
```
