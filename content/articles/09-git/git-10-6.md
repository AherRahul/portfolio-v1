# GitHub CLI (gh) — GitHub from Your Terminal

The **GitHub CLI** (`gh`) is the official command-line tool for interacting with GitHub. Instead of switching between your terminal and browser, `gh` lets you create PRs, manage issues, view workflows, and clone repositories all from the command line.

## Installation

```bash
# macOS
brew install gh

# Windows (winget)
winget install GitHub.cli
# or via Chocolatey:
choco install gh

# Linux (Debian/Ubuntu)
sudo apt install gh
```

After installing, authenticate:

```bash
gh auth login
# Follow prompts to authenticate via browser or token
```

## Core Commands

### Repository Management

```bash
# Clone a repository
gh repo clone owner/repo

# Create a new repository
gh repo create my-project --public --gitignore Node

# Fork a repository
gh repo fork owner/repo

# View repository info
gh repo view
gh repo view owner/repo --web   # Open in browser
```

### Pull Requests

```bash
# Create a PR for current branch
gh pr create --title "Add feature" --body "Description here"

# Create PR interactively
gh pr create

# List open PRs
gh pr list

# Check out a PR locally
gh pr checkout 123

# View PR status (CI checks)
gh pr status

# View a specific PR
gh pr view 123
gh pr view 123 --web

# Merge a PR
gh pr merge 123 --squash

# Review a PR
gh pr review 123 --approve
gh pr review 123 --request-changes --body "Please fix the tests"
```

### Issues

```bash
# Create an issue
gh issue create --title "Bug: login fails" --body "Details..."

# List issues
gh issue list
gh issue list --label bug --assignee @me

# View an issue
gh issue view 456

# Close an issue
gh issue close 456

# Comment on an issue
gh issue comment 456 --body "Fixed in PR #789"
```

### GitHub Actions / Workflows

```bash
# List workflow runs
gh run list

# View a specific run
gh run view 12345

# Watch a run in real-time
gh run watch

# Rerun a failed run
gh run rerun 12345

# Trigger a workflow manually
gh workflow run deploy.yml --ref main
```

### Releases

```bash
# Create a release
gh release create v1.2.0 --title "v1.2.0" --notes "Release notes"

# Upload assets to a release
gh release upload v1.2.0 dist/app.tar.gz

# List releases
gh release list

# Download release assets
gh release download v1.2.0
```

## Powerful Productivity Tips

```bash
# Create PR and open it in browser
gh pr create --web

# See all your PRs across all repos
gh pr list --author @me --repo ""

# SSH into Codespaces
gh cs ssh

# Open current repo in GitHub.com
gh repo view --web

# Create PR from current branch pointing to main
git switch -c feature/my-feature
# ... work...
git push -u origin feature/my-feature
gh pr create --base main --head feature/my-feature
```

## Configuration

```bash
# Set default editor
gh config set editor "code --wait"

# Set default git protocol
gh config set git_protocol ssh

# View current config
gh config list
```

The `gh` CLI dramatically reduces context switching, making GitHub operations feel as natural as Git operations in your terminal.
