# GitHub Actions Advanced Patterns

Building on the basics, advanced GitHub Actions usage involves reusable workflows, composite actions, matrix builds, and deployment environments to create powerful CI/CD pipelines.

## Reusable Workflows

Define a workflow once and call it from other workflows:

```yaml
# .github/workflows/reusable-test.yml
name: Reusable Test Workflow

on:
  workflow_call:
    inputs:
      node-version:
        required: false
        type: string
        default: '20'
    secrets:
      NPM_TOKEN:
        required: false

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ inputs.node-version }}
      - run: npm ci
      - run: npm test
```

Calling it from another workflow:

```yaml
# .github/workflows/ci.yml
jobs:
  call-test:
    uses: ./.github/workflows/reusable-test.yml
    with:
      node-version: '20'
    secrets:
      NPM_TOKEN: ${{ secrets.NPM_TOKEN }}
```

## Composite Actions

Create custom actions with multiple steps:

```yaml
# .github/actions/setup-app/action.yml
name: 'Setup Application'
description: 'Install and configure the application'
inputs:
  node-version:
    description: 'Node version'
    default: '20'
runs:
  using: 'composite'
  steps:
    - uses: actions/setup-node@v4
      with:
        node-version: ${{ inputs.node-version }}
        cache: 'npm'
    - run: npm ci
      shell: bash
    - run: npm run build
      shell: bash
```

Use it:

```yaml
steps:
  - uses: actions/checkout@v4
  - uses: ./.github/actions/setup-app
    with:
      node-version: '20'
```

## Complex Matrix Builds

```yaml
jobs:
  test:
    strategy:
      fail-fast: false   # Don't cancel all if one fails
      matrix:
        os: [ubuntu-latest, macos-latest, windows-latest]
        node: [18, 20, 22]
        include:
          # Add extra config for specific combo
          - os: ubuntu-latest
            node: 20
            coverage: true
        exclude:
          # Skip a specific combo
          - os: windows-latest
            node: 18
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node }}
      - run: npm ci
      - run: npm test
      - if: ${{ matrix.coverage }}
        run: npm run coverage
```

## Environment Protection Rules

```yaml
jobs:
  deploy-production:
    environment:
      name: production
      url: https://myapp.com  # Shows in GitHub UI
    steps:
      - name: Deploy
        run: ./deploy.sh
```

Set up environment protection rules in GitHub:
- **Required reviewers** — Need approval before deploying
- **Wait timer** — Delay between approval and deployment
- **Deployment branches** — Only allowed from specific branches

## Conditional Steps and Jobs

```yaml
steps:
  - name: Build for prod
    if: github.ref == 'refs/heads/main'
    run: npm run build:prod

  - name: Build for staging
    if: github.ref == 'refs/heads/develop'
    run: npm run build:staging

  - name: Notify Slack on failure
    if: failure()
    uses: 8398a7/action-slack@v3
    with:
      status: failure
```

## Caching

```yaml
- name: Cache node modules
  uses: actions/cache@v4
  with:
    path: ~/.npm
    key: ${{ runner.os }}-node-${{ hashFiles('**/package-lock.json') }}
    restore-keys: |
      ${{ runner.os }}-node-

- run: npm ci
```

## Workflow Dispatch with Inputs

Allow manual triggering with parameters:

```yaml
on:
  workflow_dispatch:
    inputs:
      environment:
        description: 'Deploy target'
        required: true
        type: choice
        options: [staging, production]
      version:
        description: 'Release version'
        required: true

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - run: echo "Deploying ${{ inputs.version }} to ${{ inputs.environment }}"
```

## Concurrency Control

Prevent multiple simultaneous deployments:

```yaml
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true  # Cancel previous runs on new push
```

## OIDC Token for Cloud Authentication

Use OIDC to authenticate with AWS/GCP/Azure without long-lived secrets:

```yaml
permissions:
  id-token: write
  contents: read

steps:
  - uses: aws-actions/configure-aws-credentials@v4
    with:
      role-to-assume: arn:aws:iam::123456789:role/my-github-role
      aws-region: us-east-1

  - name: Deploy to S3
    run: aws s3 sync dist/ s3://my-bucket/
```
