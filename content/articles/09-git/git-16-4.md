# Git Commit Signing with GPG

Signed commits use **GPG (GNU Privacy Guard)** cryptography to prove that commits were actually made by you. On GitHub, signed commits show a green **"Verified"** badge, adding trust and security to your contributions.

## Why Sign Commits?

- **Authenticity** — Anyone can set `git config user.email` to your email. Signing proves it was really you.
- **GitHub verification** — Shows the green "Verified" badge
- **Security policies** — Some organizations require signed commits
- **Open source trust** — Shows reviewers the commit is authentic

## Setting Up GPG

### 1. Generate a GPG Key

```bash
# Generate a new key (recommended: RSA 4096 or ED25519)
gpg --full-generate-key

# When prompted:
# Key type: RSA and RSA (or ECC for ED25519)
# Key size: 4096
# Expiration: 1 year (can renew)
# Real name: Your Name
# Email: your-github-email@example.com
# Passphrase: (use a strong passphrase)
```

### 2. Find Your Key ID

```bash
# List your secret keys
gpg --list-secret-keys --keyid-format=long

# Output:
# sec   rsa4096/ABC1234567890DEF 2024-01-15 [SC]
#       Full fingerprint here
# uid                 [ultimate] Jane Doe <jane@example.com>
# ssb   rsa4096/XYZ1234567890 2024-01-15 [E]

# Your Key ID is: ABC1234567890DEF
```

### 3. Configure Git to Use GPG

```bash
# Tell Git which key to use
git config --global user.signingkey ABC1234567890DEF

# Automatically sign all commits
git config --global commit.gpgsign true

# Automatically sign all tags
git config --global tag.gpgsign true

# If using gpg2
git config --global gpg.program gpg2
```

### 4. Add Your Public Key to GitHub

```bash
# Export your public key
gpg --armor --export ABC1234567890DEF
# Prints the key starting with: -----BEGIN PGP PUBLIC KEY BLOCK-----
```

Add to GitHub:
- **GitHub** → Settings → SSH and GPG keys → **New GPG key** → Paste

## Using Signed Commits

With auto-signing configured, all commits are signed automatically:

```bash
git commit -m "feat: add authentication"
# Automatically GPG-signed
```

Without auto-signing, sign manually:

```bash
git commit -S -m "feat: add authentication"  # -S = sign
git tag -s v1.0.0 -m "Signed release"        # -s = sign tag
```

## Verifying Signatures

```bash
# Verify a commit's signature
git show --show-signature abc1234

# Output includes:
# gpg: Signature made Mon Jan 15 2024...
# gpg: Good signature from "Jane Doe <jane@example.com>"
# gpg: WARNING: This key is not certified with a trusted signature!

# Verify a tag
git tag -v v1.0.0

# Log with signature status
git log --show-signature --oneline -5
```

## Troubleshooting

```bash
# Error: "gpg failed to sign the data"
# Fix: Tell GPG which TTY to use
export GPG_TTY=$(tty)
echo 'export GPG_TTY=$(tty)' >> ~/.bashrc

# On macOS, use pinentry-mac for passphrase GUI
brew install pinentry-mac
echo "pinentry-program $(which pinentry-mac)" >> ~/.gnupg/gpg-agent.conf
gpgconf --kill gpg-agent  # Restart the agent

# If Git uses wrong gpg version
git config --global gpg.program /usr/local/bin/gpg

# Skip signing just once (emergency)
git commit --no-gpg-sign -m "Emergency commit"
```

## SSH Commit Signing (Git 2.34+ Alternative)

Git 2.34+ supports SSH keys for signing (simpler than GPG):

```bash
# Tell Git to use SSH signing
git config --global gpg.format ssh
git config --global user.signingkey ~/.ssh/id_ed25519.pub

# Sign commits
git config --global commit.gpgsign true
```

Add your SSH public key to GitHub as a **"Signing Key"** (not just an "Authentication Key").

This is simpler than GPG since you likely already have an SSH key configured.
