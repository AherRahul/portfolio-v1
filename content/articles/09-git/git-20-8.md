# Git Sparse Checkout

**Sparse checkout** allows you to check out only specific directories (or files) from a large repository, leaving other parts of the working tree empty. This is valuable for monorepos or large projects where you only need to work on one component.

## Why Sparse Checkout?

In a large monorepo:

```
monorepo/
  packages/
    api/           ← You work here (100MB of files)
    web/           ← You don't need this (200MB)
    mobile/        ← You don't need this (150MB)
    data-science/  ← You don't need this (500MB)
```

Without sparse checkout: you clone everything (950MB).
With sparse checkout: you only get the parts you need.

## Setting Up Sparse Checkout (Cone Mode)

Cone mode is faster and simpler — it works with directory paths:

```bash
# Method 1: Clone with no checkout, then configure
git clone --no-checkout --filter=blob:none https://github.com/org/monorepo.git
cd monorepo

# Initialize sparse checkout in cone mode
git sparse-checkout init --cone

# Specify which directories you want
git sparse-checkout set packages/api packages/shared-utils docs/

# Now checkout
git checkout main
```

The working tree now only contains `packages/api/`, `packages/shared-utils/`, `docs/`, and top-level files.

## Modifying Sparse Patterns

```bash
# See what's currently included
git sparse-checkout list

# Add a new directory
git sparse-checkout add packages/new-lib

# Remove a directory (reset list to only what you specify)
git sparse-checkout set packages/api packages/shared-utils

# Include everything (disable sparse checkout)
git sparse-checkout disable
```

## Non-Cone Mode (Advanced)

Non-cone mode supports arbitrary pattern matching (like `.gitignore`):

```bash
git sparse-checkout init  # Without --cone

# Set patterns (saved in .git/info/sparse-checkout)
git sparse-checkout set
```

Edit `.git/info/sparse-checkout`:
```
# Include specific directories
/packages/api/
/packages/utils/
/docs/

# Include specific file types anywhere
*.md

# Exclude a specific subdirectory within one you want
!/packages/api/legacy/
```

## Combining with Partial Clone

For maximum efficiency (especially on CI):

```bash
# Partial clone: don't download file contents unless needed
# + Sparse checkout: only specific directories
git clone \
  --filter=blob:none \    # Partial clone: no blobs initially
  --no-checkout \         # Don't check out files yet
  https://github.com/org/monorepo.git

cd monorepo

git sparse-checkout init --cone
git sparse-checkout set packages/api

git checkout main   # Only downloads files for packages/api/
```

## Checking Sparse Checkout Status

```bash
# Is sparse checkout enabled?
git config core.sparseCheckout
# true / false

# What patterns are active?
git sparse-checkout list

# Check if a file path would be included
git check-attr filter .gitattributes
```

## Real-World Example: CI Pipeline

```yaml
# GitHub Actions — only check out the app you're testing
steps:
  - uses: actions/checkout@v4
    with:
      fetch-depth: 1
      sparse-checkout: |
        packages/api
        packages/shared-utils
```

This is dramatically faster than checking out the entire monorepo.
