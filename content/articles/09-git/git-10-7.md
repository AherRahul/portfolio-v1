# GitHub Discussions and Wikis

Beyond issues and PRs, GitHub provides Discussions for community conversations and Wikis for documentation. These features extend your project's collaboration capabilities beyond just code changes.

## GitHub Discussions

Discussions is a forum-like feature for conversations that don't necessarily result in a code change — questions, announcements, polls, and general project conversations.

### Enabling Discussions

Repository → Settings → General → Discussions → ✅ Enable

### Discussion Categories

Create categories to organize conversations:

| Category | Type | Use for |
|----------|------|---------|
| **Announcements** | Announcement | Release notes, important updates |
| **Q&A** | Question | Support, how-to questions |
| **Ideas** | Open-ended | Feature requests, brainstorming |
| **Show and tell** | Open-ended | What users built with your project |
| **General** | Open-ended | Everything else |
| **Polls** | Poll | Community voting |

### When to Use Discussions vs Issues

| Situation | Use |
|-----------|-----|
| Bug report | **Issue** |
| Feature request that needs implementation | **Issue** |
| "How do I...?" questions | **Discussion** |
| General ideas or RFCs | **Discussion** |
| Community announcements | **Discussion** |
| Longer open-ended conversations | **Discussion** |

### GitHub CLI for Discussions

```bash
# List discussions
gh api graphql -f query='
  query {
    repository(owner: "owner", name: "repo") {
      discussions(first: 10) {
        nodes { title url }
      }
    }
  }
'
```

### Converting Issues to Discussions

Sometimes a reported "bug" is actually a question or feature request. GitHub lets you convert issues to discussions and vice versa, keeping the conversation in the right place.

### Discussion Best Practices

- **Use categories** to help people find the right place to post
- **Mark answers** in Q&A discussions when the question is resolved
- **Pin important discussions** (e.g., roadmap, FAQ) so they're visible
- **Lock resolved discussions** to indicate they're complete
- **Reference discussions from issues** when a discussion leads to implementation

## GitHub Wiki

GitHub Wiki is a documentation platform built into every repository. It's ideal for user-facing documentation, API references, and project guides.

### Enabling and Editing Wikis

Repository → Wiki tab → Create the first page

Or clone the Wiki as a Git repo:

```bash
git clone https://github.com/user/project.wiki.git
cd project.wiki
# Edit wiki pages as Markdown files
git add .
git commit -m "Update installation guide"
git push origin master  # Wikis use 'master' by default
```

### Wiki Structure

```
Home.md            ← Landing page (always shown first)
Installation.md
Configuration.md
API-Reference.md
Contributing.md
Troubleshooting.md
```

### Sidebar and Footer

Create `_Sidebar.md` for a navigation sidebar:

```markdown
<!-- _Sidebar.md -->
## Documentation

- [Home](Home)
- [Installation](Installation)
- [Configuration](Configuration)
- [API Reference](API-Reference)

## Community

- [Contributing](Contributing)
- [Code of Conduct](Code-of-Conduct)
```

Create `_Footer.md` for a footer on every page:

```markdown
<!-- _Footer.md -->
[Back to repository](https://github.com/user/project) • [Report a bug](https://github.com/user/project/issues)
```

### Wiki vs README vs Discussions

| Content type | Where |
|-------------|-------|
| Project introduction | README |
| Concise setup guide | README |
| Detailed documentation | Wiki |
| API reference | Wiki |
| Community Q&A | Discussions |
| Announcements | Discussions |
| Bugs and features | Issues |
