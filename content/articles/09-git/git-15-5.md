# Rewriting History — git filter-repo

Sometimes you need to **rewrite your entire commit history** — removing a sensitive file that was accidentally committed, changing all commit author emails, or purging large files. These are radical operations that rewrite every commit's hash.

## git filter-repo (Modern Tool)

`git filter-repo` is the **recommended tool** for history rewriting. It replaced the older `git filter-branch` (which is slow and error-prone).

### Installation

```bash
pip install git-filter-repo
# or
brew install git-filter-repo
```

### Removing a Sensitive File from ALL History

This is the most common use case — accidentally committed a `.env` file or API key:

```bash
# 1. Clone a fresh copy (ALWAYS work on a fresh clone!)
git clone https://github.com/you/project.git project-clean
cd project-clean

# 2. Remove the file from ALL commits
git filter-repo --path secret.env --invert-paths

# The file is now gone from every commit in history

# 3. Force push to remote (required!)
git remote add origin https://github.com/you/project.git
git push origin --force --all
git push origin --force --tags
```

⚠️ **All collaborators must re-clone or rebase** after a history rewrite.

### Removing a Directory from History

```bash
git filter-repo --path node_modules --invert-paths
```

### Keeping Only Specific Files

```bash
# Keep only the src/ directory
git filter-repo --path src/ --path README.md
```

### Changing Author Name/Email

```bash
# Create a mailmap file
cat > mailmap.txt << 'EOF'
Correct Name <correct@email.com> Old Name <old@email.com>
EOF

git filter-repo --mailmap mailmap.txt
```

### Renaming a Directory in History

```bash
# Rename /src to /lib in all commits
git filter-repo --path-rename src/:lib/
```

### Removing Large Binary Files

```bash
# List the largest objects in history
git rev-list --objects --all | git cat-file --batch-check='%(objecttype) %(objectname) %(objectsize) %(rest)' | sort -k3 -rn | head -20

# Remove specific large files
git filter-repo --path large-file.psd --invert-paths
git filter-repo --path videos/ --invert-paths

# Or by size (remove files > 5MB)
git filter-repo --strip-blobs-bigger-than 5M
```

## Old Tool: git filter-branch

The older `git filter-branch` is still available but **deprecated** in favor of `git filter-repo`:

```bash
# OLD WAY: Remove a file (slow, complex)
git filter-branch --force --index-filter \
  'git rm --cached --ignore-unmatch secret.env' \
  --prune-empty --tag-name-filter cat -- --all

# Then clean up
git reflog expire --expire=now --all
git gc --prune=now --aggressive
```

## After Any History Rewrite

1. **Force push all branches and tags:**
   ```bash
   git push origin --force --all
   git push origin --force --tags
   ```

2. **Invalidate sensitive data:**
   - Rotate any exposed API keys, passwords, tokens
   - GitHub's secret scanning will alert on known patterns

3. **All collaborators must:**
   ```bash
   # Re-clone the repository
   git clone https://github.com/you/project.git
   
   # OR rebase their local work on the new history
   git fetch origin
   git rebase origin/main
   ```

4. **Update GitHub:** GitHub caches commit data — contact support if needed.

## Prevention is Better

> The best strategy is **never committing secrets in the first place**.

- Use `.gitignore` for secret files
- Use environment variables for credentials
- Use pre-commit hooks to detect secrets before commit
- Use tools like `git-secrets` or `trufflehog` for scanning
