# git status — Checking Repository State

`git status` is one of the most frequently used Git commands. It shows the current state of your working directory and staging area, telling you exactly which files are tracked, modified, staged, or untracked.

## Basic Usage

```bash
git status
```

A typical output:

```
On branch main
Your branch is up to date with 'origin/main'.

Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
        modified:   src/auth.js

Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git restore <file>..." to discard changes in working directory)
        modified:   src/payment.js

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        src/newfeature.js
```

## Understanding the Output Sections

### "Changes to be committed"
Files that have been staged (`git add`) and will be included in the next commit.

### "Changes not staged for commit"
Files that are tracked by Git and have been modified, but have NOT been staged yet.

### "Untracked files"
New files that Git doesn't know about yet (not staged, not tracked).

## File States Explained

```
┌─────────────────────────────────────────────────────────┐
│ Untracked → Staged → Committed vs Modified → Staged ... │
└─────────────────────────────────────────────────────────┘
```

| State | Description |
|-------|-------------|
| **Untracked** | New file, Git doesn't track it |
| **Modified (unstaged)** | Changes in working dir, not staged |
| **Staged** | Changes added with `git add`, ready to commit |
| **Committed** | Changes are saved in the repository |
| **Deleted** | File was removed, either staged or unstaged |

## Short Format (-s)

For a compact view:

```bash
git status -s
# or
git status --short
```

Output:
```
M  src/auth.js       ← staged modification (left column = staging, right = working)
 M src/payment.js    ← unstaged modification
?? src/newfile.js    ← untracked file
A  src/added.js      ← newly staged file
D  src/deleted.js    ← deleted file
```

Two-column status: left = staging area, right = working directory:
- `M` = Modified
- `A` = Added
- `D` = Deleted
- `R` = Renamed
- `??` = Untracked

## Branch Status

```bash
git status
# On branch feature/login
# Your branch is ahead of 'origin/feature/login' by 2 commits.
```

This shows:
- Current branch name
- How many commits ahead/behind the remote tracking branch you are

## Practical Use Cases

```bash
# Before committing — see what will be included
git status

# After git add — confirm what's staged
git status

# After a merge — check for conflicts
git status

# Check if working directory is clean (for scripts)
git status --porcelain   # Returns empty if clean
```

## Tips

```bash
# Status with branch info and short format
git status -sb

# Ignore untracked files in output
git status -u no
```

Make `git status` a habit — run it before and after every operation to stay aware of your repository's state.
