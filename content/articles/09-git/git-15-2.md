# Git Hooks

Git hooks are scripts that Git executes automatically before or after certain events, such as committing, pushing, or receiving pushed commits. They let you automate tasks, enforce policies, and integrate with other tools in your workflow.


## How Git Hooks Work

Hooks are stored in the `.git/hooks/` directory of your repository. Git comes with sample hooks (ending in `.sample`) that you can rename and modify.

```bash
ls .git/hooks/
# applypatch-msg.sample
# commit-msg.sample
# pre-commit.sample
# pre-push.sample
# prepare-commit-msg.sample
# ...
```

To activate a hook, remove the `.sample` extension and make it executable:

```bash
# On Unix/Linux/Mac:
mv .git/hooks/pre-commit.sample .git/hooks/pre-commit
chmod +x .git/hooks/pre-commit
```

Hooks can be written in any scripting language (bash, Python, Node.js, etc.).


## Client-Side Hooks

These run on your local machine during operations like committing and merging.

### pre-commit

Runs before a commit is created. If it exits with a non-zero status, the commit is aborted.

Use cases:
- Run linters (ESLint, Pylint)
- Run tests
- Check for debug statements / console.log
- Validate file formats

```bash
#!/bin/sh
# .git/hooks/pre-commit

# Run ESLint
npm run lint
if [ $? -ne 0 ]; then
  echo "Linting failed. Aborting commit."
  exit 1
fi
```

### commit-msg

Runs after you write a commit message. Receives the path to the temporary file holding the message.

Use cases:
- Enforce commit message format (conventional commits)
- Require ticket numbers

```bash
#!/bin/sh
# .git/hooks/commit-msg

commit_msg=$(cat "$1")

# Require conventional commit format: type(scope): message
if ! echo "$commit_msg" | grep -qE "^(feat|fix|docs|style|refactor|test|chore)(\(.+\))?: .+"; then
  echo "Invalid commit message format!"
  echo "Format: type(scope): message"
  echo "Types: feat, fix, docs, style, refactor, test, chore"
  exit 1
fi
```

### pre-push

Runs before `git push`. If it exits with non-zero, the push is aborted.

Use cases:
- Run the full test suite before pushing
- Prevent pushing to protected branches

```bash
#!/bin/sh
# .git/hooks/pre-push

protected_branch='main'
current_branch=$(git symbolic-ref HEAD | sed -e 's,.*/\(.*\),\1,')

if [ $protected_branch = $current_branch ]; then
  read -p "Pushing to main. Continue? (y/n) " choice
  case "$choice" in
    y|Y) exit 0 ;;
    *) exit 1 ;;
  esac
fi
```


## Server-Side Hooks

These run on the remote server when pushing commits.

### pre-receive

Runs when the server receives a push. Can reject pushes that don't meet criteria.

### update

Runs once per branch being updated. Can reject updates to specific branches.

### post-receive

Runs after the entire receive process completes. Used for notifications, CI/CD triggering.


## Sharing Hooks with Your Team

A major limitation: `.git/hooks/` is not tracked by Git (the `.git/` directory isn't committed). Solutions:

### Option 1: Husky (Node.js projects)

[Husky](https://typicode.github.io/husky/) is the most popular tool for managing Git hooks in Node.js projects:

```bash
npm install --save-dev husky
npx husky init
# Creates .husky/ directory which IS committed to the repo
```

`.husky/pre-commit`:

```bash
#!/usr/bin/env sh
. "$(dirname -- "$0")/_/husky.sh"

npm run lint
npm test
```

### Option 2: Store hooks in a directory

```bash
# Store hooks in a tracked directory
mkdir .githooks
# Create your hooks here...

# Configure git to use this directory
git config core.hooksPath .githooks
```

**Share this configuration:** Each developer needs to run the `git config` command (or add it to your project setup script).


## Summary

| Hook | When it runs | Common uses |
|------|-------------|-------------|
| `pre-commit` | Before commit | Run linter, tests |
| `commit-msg` | After commit message written | Enforce commit format |
| `prepare-commit-msg` | Before commit message editor | Insert template, add ticket number |
| `pre-push` | Before push | Run full test suite |
| `pre-receive` | On server receiving push | Enforce policies |
| `post-receive` | After server receives push | CI/CD, notifications |
