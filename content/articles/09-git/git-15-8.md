# Advanced Git Hooks

Building on the basics, advanced Git hooks let you automate complex workflows, enforce team policies, and integrate with external systems at the exact right moment in the Git lifecycle.

## Complete Hook Reference

Git hooks live in `.git/hooks/` and fire at specific points:

### Client-Side Hooks

| Hook | When It Fires | Use Case |
|------|--------------|----------|
| `pre-commit` | Before commit is created | Lint, format, tests |
| `prepare-commit-msg` | After message template | Inject branch name into message |
| `commit-msg` | After user writes message | Validate message format |
| `post-commit` | After commit is created | Notifications, CI trigger |
| `pre-rebase` | Before rebase starts | Warn if rebasing shared branches |
| `post-checkout` | After checkout | Install deps, show notes |
| `post-merge` | After merge completes | Install new deps |
| `pre-push` | Before push executes | Run tests, check what's being pushed |
| `post-rewrite` | After amend/rebase | Update references |

### Server-Side Hooks

| Hook | When It Fires | Use Case |
|------|--------------|----------|
| `pre-receive` | Before push is accepted | Enforce branch policies |
| `update` | For each ref being updated | Per-branch validation |
| `post-receive` | After push is accepted | Deploy, notify, CI trigger |

## Useful Hook Examples

### prepare-commit-msg: Auto-Insert Branch Name

```bash
#!/bin/sh
# .git/hooks/prepare-commit-msg

COMMIT_MSG_FILE=$1
COMMIT_SOURCE=$2

# Get current branch name
BRANCH=$(git branch --show-current)

# Extract issue ID from branch name (e.g., feature/PROJ-123-login → PROJ-123)
ISSUE=$(echo "$BRANCH" | grep -oE '[A-Z]+-[0-9]+')

# Prepend to commit message if not already there
if [ -n "$ISSUE" ] && ! grep -q "$ISSUE" "$COMMIT_MSG_FILE"; then
  sed -i.bak -e "1s/^/$ISSUE: /" "$COMMIT_MSG_FILE"
fi
```

### post-checkout: Install Dependencies When Lockfile Changes

```bash
#!/bin/sh
# .git/hooks/post-checkout

PREVIOUS_HEAD=$1
NEW_HEAD=$2
BRANCH_CHECKOUT=$3

if [ "$BRANCH_CHECKOUT" -eq 1 ]; then
  # Check if package-lock.json changed
  CHANGED=$(git diff --name-only $PREVIOUS_HEAD $NEW_HEAD -- package-lock.json)
  if [ -n "$CHANGED" ]; then
    echo "📦 package-lock.json changed — running npm install..."
    npm install
  fi
fi
```

### post-merge: Update Deps After Pull

```bash
#!/bin/sh
# .git/hooks/post-merge

CHANGED=$(git diff-tree -r --name-only --no-commit-id ORIG_HEAD HEAD)

if echo "$CHANGED" | grep -q "package-lock.json"; then
  echo "📦 Dependencies changed — running npm install..."
  npm install
fi

if echo "$CHANGED" | grep -q "Gemfile.lock"; then
  echo "💎 Bundle changed — running bundle install..."
  bundle install
fi
```

### pre-rebase: Warn About Shared Branches

```bash
#!/bin/sh
# .git/hooks/pre-rebase

UPSTREAM=$1
BRANCH=$2

# Warn if trying to rebase main or develop
PROTECTED="main develop master"
for BRANCH in $PROTECTED; do
  if git branch --contains HEAD | grep -q "^. $BRANCH$"; then
    echo "⚠️  WARNING: You're about to rebase a protected branch!"
    echo "   This will rewrite shared history."
    echo "   Press Enter to continue or Ctrl+C to abort."
    read
  fi
done
```

## Using Husky for Team Hooks

Individual `.git/hooks/` aren't shared via Git. Husky lets you share hooks:

```bash
npm install --save-dev husky

# Enable Husky
npx husky init

# Create hooks
echo "npm run lint" > .husky/pre-commit
echo "npm test" >> .husky/pre-push

# Make executable
chmod +x .husky/pre-commit .husky/pre-push
```

`package.json`:

```json
{
  "scripts": {
    "prepare": "husky"
  }
}
```

Now every developer who runs `npm install` gets the hooks automatically.

## Lint-Staged (Only Lint Changed Files)

```bash
npm install --save-dev lint-staged
```

```json
// package.json
{
  "lint-staged": {
    "*.{js,ts}": ["eslint --fix", "prettier --write"],
    "*.{css,scss}": ["prettier --write"],
    "*.md": ["prettier --write"]
  }
}
```

`.husky/pre-commit`:
```bash
npx lint-staged
```

This runs lint only on the files you're about to commit — much faster than linting the entire codebase.

## Server-Side Hooks (Self-Hosted Git)

For GitHub Enterprise or self-hosted GitLab/Gitea:

```bash
# On the server, in the bare repository
# .git/hooks/pre-receive or hooks/pre-receive

#!/bin/sh
while read OLDREV NEWREV REFNAME; do
  BRANCH=$(echo "$REFNAME" | sed 's|refs/heads/||')
  
  # Block direct pushes to main
  if [ "$BRANCH" = "main" ]; then
    echo "❌ Direct pushes to main are not allowed."
    echo "   Please use a pull request."
    exit 1
  fi
done
exit 0
```
