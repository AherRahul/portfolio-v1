# Git Packfiles — Efficient Storage

As your repository grows, Git uses **packfiles** to efficiently store objects together using compression and delta encoding. Understanding packfiles helps you understand why Git repositories are small and fast.

## Loose Objects vs. Packed Objects

Initially, every Git object (blob, tree, commit, tag) is stored as an individual compressed file in `.git/objects/`:

```
.git/objects/
  ab/
    c1234567890...   ← One file per object ("loose object")
  de/
    f5678...
```

This works for small repositories, but becomes inefficient with thousands of objects. Git solves this with **packfiles**.

## What is a Packfile?

A packfile groups many objects together in a single binary file:

```
.git/objects/pack/
  pack-abc123.pack   ← The packed objects
  pack-abc123.idx    ← Index for fast lookup
```

Packfiles use **delta compression**: instead of storing complete copies of similar objects, they store the differences (deltas) between them. For example, if you change one line in a 1000-line file, Git stores the full version once and just the delta for subsequent versions.

## When Git Creates Packfiles

Git automatically packs objects when:

1. **You run `git gc`** (garbage collection)
2. **You clone a repository** (the remote sends a packfile)
3. **You push** (Git creates a packfile for transfer)
4. **Too many loose objects exist** (Git auto-runs `git gc` around 6700+ loose objects)

## Manual Packing

```bash
# Pack all loose objects
git gc

# More aggressive repacking
git gc --aggressive

# Just pack loose objects into packfiles
git pack-objects .git/objects/pack/new-pack < /dev/stdin

# Verify pack integrity
git fsck

# Count objects
git count-objects -v
```

Example output of `git count-objects -v`:

```
count: 3           ← Loose objects
size: 12           ← Size of loose objects (KB)
in-pack: 1847      ← Objects in packfiles
packs: 1           ← Number of packfiles
size-pack: 423     ← Size of packfiles (KB)
prune-packable: 0
garbage: 0
size-garbage: 0
```

## Delta Compression

Git uses delta compression to reduce packfile size:

```
Version 1: [Full file — 10KB]
Version 2: [Delta from V1 — 100 bytes]
Version 3: [Delta from V2 — 200 bytes]
```

Git is clever: it may delta-compress objects that are similar even if they're not in a parent-child relationship (e.g., same filename in different commits, or similar content).

## Inspecting Packfiles

```bash
# List objects in a packfile
git verify-pack -v .git/objects/pack/pack-*.idx

# Output:
# abc1234 commit 280 203 100
# def5678 tree   164 131 303
# ...

# Format: hash type size offset
```

## gc and git prune

`git gc` also removes unreachable objects (e.g., commits dangling after a reset or rebase):

```bash
git gc                    # Standard cleanup
git gc --prune=now        # Remove unreachable objects immediately (default: 2 weeks grace)
git prune                 # Manually remove unreachable loose objects
git reflog expire --all   # Expire all reflog entries
```

## Bitmap Index

For performance, Git can create **bitmap indexes** for fast object reachability queries:

```bash
git repack -a -d --write-bitmap-index
```

Bitmaps dramatically speed up operations like `git clone` and `git fetch` for large repositories.
