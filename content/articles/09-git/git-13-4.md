# Advanced Rebase Techniques

Beyond the basic `git rebase main`, interactive rebase lets you sculpt your commit history into a clean, coherent narrative before merging.

## Launching Interactive Rebase

```bash
# Rebase last N commits
git rebase -i HEAD~5

# Rebase all commits since branching from main
git rebase -i main

# Opens an editor with a list of commits:
# pick abc1234 Add login form
# pick def5678 Fix typo in button
# pick g9h0i1j Add login validation
# pick k2l3m4n WIP: working on error handling
# pick p5q6r7s Complete error handling and tests
```

## Interactive Rebase Commands

Change `pick` to any of these commands:

```
pick    = Include this commit as-is
r/reword = Include, but edit the commit message
e/edit  = Include, but stop here to amend the commit
s/squash = Merge into previous commit (combine messages)
f/fixup  = Like squash, but discard this commit's message
d/drop   = Remove this commit entirely
x/exec  = Run a shell command after this commit
b/break = Stop here (useful for debugging)
```

## Squashing Commits

Clean up a messy feature branch before merging:

```
# Before: 5 messy commits
pick abc1234 Add login form
pick def5678 Fix typo
pick g9h0i1j Fix lint issues
pick k2l3m4n Add validation
pick p5q6r7s Add tests

# Change to:
pick abc1234 Add login form
f def5678 Fix typo       ← fixup (merge, discard message)
f g9h0i1j Fix lint issues ← fixup (merge, discard message)
s k2l3m4n Add validation   ← squash (merge, keep message)
s p5q6r7s Add tests         ← squash (merge, keep message)
```

Result: 2 clean commits instead of 5.

## Reordering Commits

Simply change the order in the editor:

```
# Before:
pick abc1234 Add login form
pick def5678 Add logout functionality
pick g9h0i1j Add login tests

# After reordering:
pick abc1234 Add login form
pick g9h0i1j Add login tests   ← moved after the form
pick def5678 Add logout functionality
```

## Editing a Commit

Fix a commit in the middle of history:

```
# In interactive rebase, mark with 'edit':
edit abc1234 Add login form   ← will pause here

# Git stops after applying abc1234
# Make your additional changes:
git add forgotten-validation.js
git commit --amend --no-edit

# Or add a new separate commit:
git add extra-fix.js
git commit -m "fix: add missing null check"

# Continue the rebase
git rebase --continue
```

## Splitting a Commit

Too much in one commit? Split it:

```
# Mark the commit with 'edit'
edit abc1234 Add login form and validation and tests (too much!)

# When git stops:
git reset HEAD~   # Undo the commit, keep changes staged

# Now commit the pieces separately
git add src/login-form.js
git commit -m "feat: add login form"

git add src/validation.js
git commit -m "feat: add login validation"

git add tests/login.test.js
git commit -m "test: add login form tests"

git rebase --continue
```

## Exec: Run Commands Between Commits

Test that each commit builds successfully:

```
pick abc1234 Add login form
exec npm test
pick def5678 Add validation  
exec npm test
pick g9h0i1j Add tests
exec npm test
```

Or run it automatically:

```bash
git rebase -i --exec "npm test" main
# Automatically inserts exec steps after each commit
```

If any `npm test` fails, rebase stops for you to fix.

## The --autosquash Workflow

Prepare fixup commits as you work, then rebase:

```bash
# Normal development
git commit -m "feat: add login form"

# Later, you realize you need to fix something in that commit
git add fix.js
git commit --fixup=HEAD~3   # "fixup! feat: add login form"

# At the end, clean up with autosquash
git rebase -i --autosquash main
# Automatically reorders and squashes fixup commits!
```

## Force Push Considerations

After any interactive rebase, your history is rewritten:

```bash
# Push rewritten history to your remote branch
git push --force-with-lease origin feature/my-branch

# --force-with-lease is safer than --force:
# It fails if someone else pushed to the branch since you last fetched
# Thus preventing accidentally overwriting their work
```
