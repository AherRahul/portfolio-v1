# Testing in Git Workflows

Integrating testing into your Git workflow ensures that every commit, branch, and merge maintains code quality. The relationship between Git and testing is fundamental to modern CI/CD.

## The Test-Commit-Push Cycle

A healthy development loop:

```
Write test → Write code → Run tests → Commit → Push → CI runs → Merge
```

**Test-Driven Development (TDD) with Git:**

```bash
# 1. Write a failing test first
git add tests/payment.test.js
git commit -m "test: add test for declined card handling"

# 2. Write code to make it pass
git add src/payment.js
git commit -m "feat: handle declined card error"

# 3. Refactor if needed
git commit -m "refactor: extract validation to helper"
```

## Testing at Each Git Boundary

| Git Action | Tests to Run |
|-----------|-------------|
| Pre-commit (hook) | Lint, type check, related unit tests |
| PR opened | Full unit + integration tests |
| PR merging | Full test suite + E2E |
| Tag push | Full test suite + E2E + performance |
| Deploy | Smoke tests post-deploy |

## Pre-commit Test Strategy

Don't run ALL tests pre-commit — too slow and disruptive:

```bash
# .husky/pre-commit
#!/bin/sh

# Run ONLY tests related to changed files
npx jest --testPathPattern="$(git diff --cached --name-only | tr '\n' '|' | sed 's/|$//')" --passWithNoTests

# Type check
npx tsc --noEmit

# Lint staged files only (via lint-staged)
npx lint-staged
```

## PR Testing Strategy

Run the full test suite on every PR:

```yaml
# .github/workflows/pr-tests.yml
name: PR Tests

on:
  pull_request:
    types: [opened, synchronize, reopened]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # Full history for coverage comparison

      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - run: npm ci

      - name: Run Unit Tests with Coverage
        run: npm test -- --coverage

      - name: Report Coverage
        uses: codecov/codecov-action@v4

      - name: Integration Tests
        run: npm run test:integration
        env:
          DATABASE_URL: ${{ secrets.TEST_DB_URL }}
```

## Branch Testing Strategies

### Feature Branch: Unit Tests

```bash
# When working on a feature branch
# Only run tests related to your changes
npm test -- --watch    # Watch mode during development
npx jest auth.test.js  # Run specific test file
```

### Main Branch: Full Suite

```yaml
jobs:
  full-suite:
    if: github.ref == 'refs/heads/main'
    steps:
      - run: npm test -- --coverage --runInBand
      - run: npm run test:e2e
      - run: npm run test:performance
```

## Test Naming for Git Integration

Structure test names to link to Git issues/features:

```javascript
// Tests clearly linked to a feature
describe('User Authentication', () => {
  describe('Google OAuth (#234)', () => {
    it('redirects to Google consent page');
    it('handles OAuth callback with valid code');
    it('handles OAuth callback with invalid code');
  });
  
  describe('Rate Limiting (#456)', () => {
    it('blocks after 5 failed attempts');
    it('resets count after timeout');
  });
});
```

## Preventing Test Regressions

```yaml
# Mandate test coverage thresholds
- name: Check Coverage
  run: |
    COVERAGE=$(npm test -- --coverage --coverageReporters=text | grep "All files" | awk '{print $10}' | tr -d '%')
    if [ "$COVERAGE" -lt 80 ]; then
      echo "Coverage $COVERAGE% is below 80% threshold"
      exit 1
    fi
```

## Git-Based Test Selection

Only run tests affected by changed files (e.g., Jest's `--changedSince`):

```bash
# Run only tests changed since main
npx jest --changedSince=main

# Run only tests changed since a specific commit
npx jest --changedSince=abc1234

# In CI for PRs:
npx jest --changedSince=origin/main
```

This dramatically speeds up feedback loops for large test suites.
