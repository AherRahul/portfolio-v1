# GitHub Projects and Milestones

GitHub Projects and Milestones connect your Git workflow to project management, keeping code work and planning in one place.

## GitHub Projects (New Projects Experience)

GitHub Projects is a flexible, spreadsheet-like project management tool that links to your repository's issues and PRs.

### Creating a Project

1. Navigate to your repository (or organization)
2. Click **"Projects"** tab → **"New project"**
3. Choose a template:
   - **Board** — Kanban/Scrum style
   - **Table** — Spreadsheet with custom fields
   - **Roadmap** — Timeline view

### Board View (Kanban)

A typical software development board:

```
📋 Backlog | 🔄 In Progress | 👀 In Review | ✅ Done
   #234       #245              #240           #238
   #235       #246              #241           #239
   #236
```

Configure automated status moves:
- When a PR is opened → Move linked issue to "In Progress"
- When PR is merged → Move to "Done"
- When PR is closed → Move to "Done"

### Custom Fields

Add metadata to issues and PRs:
- **Priority:** Urgent, High, Medium, Low
- **Estimate:** Story points (1, 2, 3, 5, 8, 13)
- **Sprint:** Sprint 1, Sprint 2, etc.
- **Team:** Frontend, Backend, DevOps
- **Due Date:** Target completion date

### Roadmap View

Plan features across time:

```bash
📅 Q1 2024        Q2 2024         Q3 2024         Q4 2024
│                 │               │               │
├─ Auth System    ├─ Dashboard    ├─ Mobile App   ├─ Scaling
├─ Payment Gate.  ├─ Analytics    ├─ Offline Mode ├─ Enterprise
└─ Email Notifs   └─ Reports      └─ Push Notifs  └─ Multi-tenant
```

### Filtering and Grouping

```
Filter: is:open assignee:@me
Group by: Sprint
Sort: Priority
```

## GitHub Milestones

Milestones group issues and PRs toward a specific goal (like a release version or sprint).

### Creating a Milestone

Repository → Issues → Milestones → New milestone

```
Title: v2.0.0 Release
Due date: 2024-03-15
Description: All features and fixes for the v2.0 release
```

### Using Milestones

```bash
# Assign via GitHub UI when creating issues
# Or via CLI
gh issue create --title "Add dark mode" --milestone "v2.0.0 Release"

# Close milestone when done
gh api repos/{owner}/{repo}/milestones/{number} -X PATCH -f state=closed
```

### Milestone Overview

GitHub shows:
- **Completion percentage** — How many issues are closed
- **Open/Closed issue count**
- **Due date and days remaining**
- **Issues list** — Sort by priority, assignee, etc.

This makes milestones perfect for release planning and sprint tracking.

### Milestones for Release Planning

```
v1.5.0 Release (Due: Jan 15)
  ██████████░░░░░░░░░░ 60% complete
  12 open / 18 closed
  
Issues:
  ✅ Add email verification
  ✅ Fix logout not working
  🔄 Add OAuth login (#234)
  📋 Mobile responsive header (#237)
  📋 Dark mode support (#239)
```

### Best Practice: Milestone + Project Combination

For structured teams:
- **Milestones** for time-bound goals (sprint, release)
- **Projects** for tracking status (backlog, in progress, done)
- **Labels** for categorization (bug, feature, priority)
- **Assignees** for ownership

Together these provide full visibility into team progress without leaving GitHub.
