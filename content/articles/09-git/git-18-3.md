# gitignore Best Practices

A `.gitignore` file tells Git which files and directories to ignore when tracking changes. Proper use of `.gitignore` keeps your repository clean, secure, and free from clutter.


## How .gitignore Works

Git ignores files matching patterns in `.gitignore`. The file lives in the repository root (or in subdirectories for more targeted rules).

```bash
# Create .gitignore in the repository root
touch .gitignore
```


## Pattern Syntax

```gitignore
# This is a comment
node_modules/        # Ignore entire directory
*.log                # Ignore all .log files
!important.log       # Exception: track this log
build/               # Ignore build directory
/dist/               # Only ignore dist in root (not subdirs)
**/*.env             # Ignore .env files anywhere in tree
temp-*               # Ignore files starting with temp-
```

Key patterns:
- `*` — Matches any string (except `/`)
- `**` — Matches any string including `/`
- `?` — Matches any single character
- `[abc]` — Matches characters in brackets
- `!pattern` — Negation: track this despite earlier ignore rule


## What to Ignore

### Always Ignore

```gitignore
# Dependencies
node_modules/
vendor/
.venv/
__pycache__/
*.pyc
*.pyo

# Build outputs
dist/
build/
out/
.next/
.nuxt/

# Environment and secrets
.env
.env.local
.env.*.local
*.pem
*.key
secrets.json
credentials.json

# OS generated files
.DS_Store          # macOS
.DS_Store?
._*
.Spotlight-V100
.Trashes
Thumbs.db          # Windows
desktop.ini

# Editor/IDE
.vscode/
.idea/
*.swp              # Vim swap files
*.swo
.project
.classpath
.settings/

# Testing
coverage/
.nyc_output/
junit.xml

# Logs
*.log
logs/
npm-debug.log*
yarn-debug.log*
yarn-error.log*
```


## What NOT to Ignore

```gitignore
# These should always be committed:
# - Source code files (.js, .py, .go, etc.)
# - Configuration templates (.env.example, config.sample.json)
# - Documentation (README.md, docs/)
# - Test files (*.test.js, *.spec.py)
# - Database migrations
# - The .gitignore file itself
```

Always commit `.env.example` (template with dummy values) but never `.env` (real secrets).


## Global .gitignore

Set up a global `.gitignore` for files you always want to ignore across all projects (like OS files and editor files):

```bash
# Create a global gitignore
touch ~/.gitignore_global

# Configure git to use it
git config --global core.excludesFile ~/.gitignore_global
```

`~/.gitignore_global`:

```gitignore
# macOS
.DS_Store
.AppleDouble
.LSOverride

# Windows
Thumbs.db
ehthumbs.db

# VS Code
.vscode/
*.code-workspace

# JetBrains
.idea/
*.iml

# Vim
*.swp
*.swo
[._]*.s[a-v][a-z]
[._]*.sw[a-p]
```


## When You Already Committed a File

If you accidentally committed a file that should be ignored:

```bash
# Remove from tracking but keep the file locally
git rm --cached file.env
git rm --cached -r node_modules/

# Add to .gitignore
echo "file.env" >> .gitignore
echo "node_modules/" >> .gitignore

# Commit the removal
git commit -m "Remove accidentally committed files"
```


## Project Templates

Use [gitignore.io](https://gitignore.io) or GitHub's template generator to create language/framework-specific `.gitignore` files:

```bash
# Via curl (for Node.js + macOS + Windows)
curl -L https://gitignore.io/api/node,macos,windows,visualstudiocode > .gitignore
```


## Debugging .gitignore

```bash
# Check why a specific file is ignored
git check-ignore -v filename.env

# List all ignored files
git ls-files --others --ignored --exclude-standard

# List tracked files that match gitignore patterns
git ls-files --ignored --exclude-standard
```
