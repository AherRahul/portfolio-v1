# Common Git Mistakes and How to Fix Them

Even experienced developers make Git mistakes. The good news: Git almost always has a way to undo what you've done. Here are the most common mistakes and their fixes.

## Mistake 1: Committed to the Wrong Branch

```bash
# You committed to main instead of your feature branch
git log --oneline -3
# abc1234 My feature commit ← Should be on feature branch!
# def5678 Previous commit

# Fix: Create the feature branch pointing to the commit
git branch feature/my-feature   # Creates branch at current HEAD

# Move main back (remove the commit from main)
git reset --hard HEAD~1   # Local only
# or
git reset --hard origin/main   # If not yet pushed

# Now work on the feature branch
git switch feature/my-feature
```

## Mistake 2: Committed Sensitive Data (API Key, Password)

```bash
# Option A: If not yet pushed — amend
git rm --cached config/secrets.env
echo "config/secrets.env" >> .gitignore
git commit --amend --no-edit

# Option B: If pushed — use git filter-repo
git filter-repo --path config/secrets.env --invert-paths
git push --force-with-lease origin --all

# ALWAYS rotate the exposed credentials regardless!
```

## Mistake 3: Made a Bad Commit (Not Yet Pushed)

```bash
# Fix the last commit completely
git reset --soft HEAD~1   # Undo commit, keep changes staged
git reset --mixed HEAD~1  # Undo commit, keep changes unstaged
git reset --hard HEAD~1   # Undo commit, DISCARD changes (dangerous!)

# Or just amend if it's a small fix
git commit --amend -m "Better commit message"
```

## Mistake 4: Made a Bad Commit (Already Pushed)

```bash
# Don't rewrite history on shared branches!
# Instead, create a revert commit

git revert abc1234
git push origin main   # Safe to push — new commit undoing the bad one
```

## Mistake 5: Accidentally Deleted a Branch

```bash
# Find the last commit on the deleted branch
git reflog | grep "branch-name"
# Or look for recent commits
git reflog --all | head -30

# Found it: abc1234
# Recreate the branch
git branch my-branch abc1234
git switch my-branch
```

## Mistake 6: Merged the Wrong Branch

```bash
# Undo the merge (if not yet pushed)
git reset --hard ORIG_HEAD

# If already pushed, revert the merge commit
git log --oneline -5   # Find the merge commit hash
git revert -m 1 abc1234  # -m 1 = revert to first parent (mainline)
git push origin main
```

## Mistake 7: Git Pull Caused a Mess

```bash
# To undo: go back to where you were before the pull
git reset --hard ORIG_HEAD  # ORIG_HEAD is set by git pull
```

## Mistake 8: Rebased the Wrong Branch

```bash
# Rebase went wrong — ORIG_HEAD saves you!
git reset --hard ORIG_HEAD
```

## Mistake 9: Detached HEAD State

```bash
# git status shows "HEAD detached at abc1234"

# Option A: Go back to your branch
git switch main   # or whatever branch you want

# Option B: Keep your work — create a branch from this commit
git switch -c my-new-branch
# Now commits won't be lost
```

## Mistake 10: git add . Staged Too Much

```bash
# Unstage everything
git reset HEAD

# Unstage a specific file
git restore --staged filename.js

# Or unstage all
git restore --staged .
```

## Mistake 11: Forgot to Include a File in Last Commit

```bash
git add forgotten-file.js
git commit --amend --no-edit   # Adds file to last commit
```

## Mistake 12: Hard Reset Lost My Work

```bash
# Commits aren't gone — they're in the reflog!
git reflog   # Find the commit before the reset

# Restore
git reset --hard abc1234   # The commit from before the bad reset
# or
git checkout abc1234        # In detached HEAD state
git switch -c recovery      # Save it in a branch
```

## Prevention: Know Before You Act

```bash
# Before any dangerous operation:
git status          # Know where you are
git log --oneline -5   # Know what's committed
git stash           # Save work-in-progress

# The two commands to never fear:
git reflog          # Shows all recent HEAD movements
git stash list      # Shows all saved stashes
```
