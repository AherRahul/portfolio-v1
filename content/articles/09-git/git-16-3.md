# Git Credential Management

Properly managing credentials for Git remote operations (push, pull, clone) is essential for security and convenience. Git supports several authentication methods: SSH keys, HTTPS with tokens, and OS-level credential managers.

## SSH Keys (Recommended)

SSH key authentication is the most secure and convenient method for GitHub/GitLab. Once set up, you never need to enter a password.

### Generating an SSH Key

```bash
# Generate a new ED25519 key (modern, recommended)
ssh-keygen -t ed25519 -C "your_email@example.com"

# Or RSA key (older, widely compatible)
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

# Default location: ~/.ssh/id_ed25519 (private) and ~/.ssh/id_ed25519.pub (public)
```

### Adding to SSH Agent

```bash
# Start the agent
eval "$(ssh-agent -s)"

# Add your key
ssh-add ~/.ssh/id_ed25519

# On macOS, persist across reboots
ssh-add --apple-use-keychain ~/.ssh/id_ed25519
```

### Add Public Key to GitHub

```bash
# Copy public key to clipboard
cat ~/.ssh/id_ed25519.pub
```

On GitHub: **Settings → SSH and GPG keys → New SSH key** → Paste.

### Test Connection

```bash
ssh -T git@github.com
# Hi username! You've successfully authenticated.
```

### Clone with SSH

```bash
git clone git@github.com:username/repository.git
```

### Switch Existing Repo from HTTPS to SSH

```bash
git remote set-url origin git@github.com:username/repository.git
```

## HTTPS with Personal Access Tokens

GitHub no longer accepts passwords for HTTPS. Use **Personal Access Tokens (PAT)** instead:

1. GitHub → **Settings → Developer settings → Personal access tokens**
2. Generate token with `repo` scope
3. Copy the token (shown only once!)

```bash
# Clone with HTTPS (you'll be prompted for credentials)
git clone https://github.com/username/repository.git
# Username: your-github-username
# Password: YOUR_PERSONAL_ACCESS_TOKEN (not your account password!)
```

## Credential Helpers

Avoid entering credentials every time using credential helpers:

```bash
# Memory cache (15 minutes default)
git config --global credential.helper cache

# Cache for 1 hour
git config --global credential.helper 'cache --timeout=3600'

# Store permanently in plain text file (~/.git-credentials) - less secure
git config --global credential.helper store

# macOS Keychain (recommended for macOS)
git config --global credential.helper osxkeychain

# Windows Credential Manager
git config --global credential.helper manager-core

# Git Credential Manager (cross-platform - best for HTTPS)
git config --global credential.helper manager
```

## Multiple GitHub Accounts with SSH

Managing multiple accounts (personal + work) on the same machine:

```bash
# Generate a separate key for work
ssh-keygen -t ed25519 -C "work@company.com" -f ~/.ssh/id_ed25519_work

# Configure ~/.ssh/config
cat >> ~/.ssh/config << 'EOF'
# Personal GitHub
Host github.com
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_ed25519

# Work GitHub
Host github-work
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_ed25519_work
EOF
```

Now use different hosts for each account:

```bash
# Personal repos
git clone git@github.com:personaluser/repo.git

# Work repos (use the alias "github-work")
git clone git@github-work:company/repo.git

# Update remote for existing work repo
git remote set-url origin git@github-work:company/repo.git
```

## GPG Commit Signing

For verified commits (shows "Verified" badge on GitHub):

```bash
# List GPG keys
gpg --list-secret-keys --keyid-format=long

# Configure Git to use your key
git config --global user.signingkey YOUR_KEY_ID

# Sign all commits by default
git config --global commit.gpgsign true

# Sign a single commit manually
git commit -S -m "Verified commit message"
```

Add your GPG public key to GitHub the same way as SSH keys.
