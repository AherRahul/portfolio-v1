# git diff — Viewing Changes

`git diff` shows the differences between various states of your repository. It's an essential command for reviewing changes before staging or committing.

## Understanding Diff Output

```diff
diff --git a/src/auth.js b/src/auth.js
index abc1234..def5678 100644
--- a/src/auth.js
+++ b/src/auth.js
@@ -10,7 +10,9 @@ function login(user, pass) {
   if (!user || !pass) return false;
+  // Validate email format
+  if (!isValidEmail(user)) return false;
   const hash = hashPassword(pass);
-  return compare(hash, stored);
+  return compare(hash, stored) && !user.banned;
 }
```

- Lines starting with `-` (red): **Removed** lines
- Lines starting with `+` (green): **Added** lines
- Lines with no prefix (white): **Context** lines (unchanged, shown for reference)
- `@@ -10,7 +10,9 @@` — "hunk header" showing line numbers in old (10,7) and new (10,9) file

## Common diff Commands

```bash
# Diff: working directory vs staging area (unstaged changes)
git diff

# Diff: staging area vs last commit (staged changes)
git diff --staged
git diff --cached   # Same as --staged

# Diff: working directory + staging vs last commit (all changes)
git diff HEAD

# Diff between two commits
git diff abc1234 def5678

# Diff between two branches
git diff main feature/login

# Diff for a specific file
git diff src/auth.js

# Diff between a commit and current
git diff HEAD~3 HEAD
```

## Comparing Branches

```bash
# What changed in feature vs main?
git diff main..feature/login

# What's in feature that's not in main? (three-dot notation)
git diff main...feature/login
```

Two dots (`..`): Compare tips of both branches directly.
Three dots (`...`): Compare from their common ancestor.

## Useful Options

```bash
# Show only file names (not full diff)
git diff --name-only

# Show file names with status (M=modified, A=added, D=deleted)
git diff --name-status

# Show stats (how many lines changed per file)
git diff --stat

# Word-by-word diff (useful for prose)
git diff --word-diff

# Ignore whitespace changes
git diff -w
git diff --ignore-all-space

# Color output
git diff --color

# Compact diff showing only what changed within a line
git diff --color-words
```

## Diff with External Tools

Configure a visual diff tool:

```bash
# Set VS Code as diff tool
git config --global diff.tool vscode
git config --global difftool.vscode.cmd 'code --wait --diff $LOCAL $REMOTE'

# Launch git difftool
git difftool
```

## Practical Workflow

```bash
# 1. Work on some files…

# 2. Review ALL unstaged changes before staging
git diff

# 3. Stage what you like
git add src/auth.js

# 4. Review what you've staged (confirms what will be committed)
git diff --staged

# 5. Quick sanity check on stats
git diff --stat HEAD   # Full picture of all changes
```

Regular use of `git diff` ensures you always know exactly what's going into your next commit.
