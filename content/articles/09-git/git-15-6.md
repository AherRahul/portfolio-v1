# git archive — Exporting Repository Contents

`git archive` creates an archive of a repository at a specific point in history — a clean export without the `.git/` directory. It's useful for creating release packages, deployment archives, and exporting code to users.

## Basic Usage

```bash
# Create a tar archive of the current HEAD
git archive --format=tar HEAD > project.tar

# Create a compressed tar.gz archive
git archive --format=tar.gz HEAD > project.tar.gz

# Create a zip archive
git archive --format=zip HEAD > project.zip

# Specify output with -o (auto-detects format from extension)
git archive -o project.tar.gz HEAD
git archive -o release.zip HEAD
```

## Archiving a Specific Version

```bash
# Archive a specific commit
git archive -o release.tar.gz abc1234

# Archive a tagged release
git archive -o v1.2.0.tar.gz v1.2.0

# Archive a specific branch
git archive -o feature.tar.gz feature/new-ui

# Archive with a prefix (adds directory inside archive)
git archive --prefix="myproject-v1.2.0/" -o v1.2.0.tar.gz v1.2.0
```

The `--prefix` adds a top-level directory inside the archive, so the contents are at `myproject-v1.2.0/src`, `myproject-v1.2.0/README.md`, etc. This is the standard GitHub behaviour.

## Archiving a Subdirectory

```bash
# Archive only the src/ directory
git archive HEAD src/ > src-only.tar.gz

# Archive specific files
git archive HEAD README.md package.json src/ > release.tar.gz
```

## The export-ignore Attribute

Files marked with `export-ignore` in `.gitattributes` are excluded from archives:

```gitattributes
# .gitattributes
.gitignore      export-ignore
.gitattributes  export-ignore
tests/          export-ignore
.github/        export-ignore
*.test.js       export-ignore
```

This is how you prevent development-only files (tests, CI configs, etc.) from appearing in release packages.

## Creating Releases

A typical release workflow:

```bash
# Tag the release
git tag -a v1.2.0 -m "Release v1.2.0"

# Create release archive with prefix
git archive \
  --prefix="myproject-v1.2.0/" \
  --format=tar.gz \
  -o myproject-v1.2.0.tar.gz \
  v1.2.0

# Also create zip
git archive \
  --prefix="myproject-v1.2.0/" \
  --format=zip \
  -o myproject-v1.2.0.zip \
  v1.2.0

# Upload to GitHub release
gh release create v1.2.0 \
  --title "v1.2.0" \
  myproject-v1.2.0.tar.gz \
  myproject-v1.2.0.zip
```

## Remote Archive (from GitHub/GitLab)

You can also create archives from a remote without cloning:

```bash
git archive --remote=git@github.com:owner/repo.git HEAD > remote-archive.tar
```

Note: This requires the remote to allow archive access (GitHub restricts this for private repos).

## Comparing git archive vs Other Methods

| Method | Includes .git | Clean export | Respects export-ignore |
|--------|--------------|--------------|------------------------|
| `git archive` | ❌ No | ✅ Yes | ✅ Yes |
| `cp -r` / zip | ✅ Yes | ❌ No | ❌ No |
| GitHub "Download ZIP" | ❌ No | ✅ Yes | ✅ Yes |
| `pip install` / npm pack | ❌ No | ✅ Yes | Varies |
