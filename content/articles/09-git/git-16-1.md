# Git Aliases

Git aliases allow you to create shortcut names for frequently used Git commands, or even create entirely new commands. They're one of the best ways to improve your daily Git productivity.


## What are Git Aliases?

A Git alias is a custom shortcut for a Git command. For example, instead of typing `git status`, you could type `git st`. Instead of the long `git log --oneline --graph --all --decorate`, you could type `git lg`.


## Creating Aliases

Use `git config` to create aliases:

```bash
# Create a global alias (available in all repositories)
git config --global alias.st status

# Now you can use:
git st  # Same as: git status
```

Aliases are stored in `~/.gitconfig` (global) or `.git/config` (local):

```ini
[alias]
    st = status
    co = checkout
    br = branch
    ci = commit
```


## Commonly Useful Aliases

### Basic Shortcuts

```bash
git config --global alias.st status
git config --global alias.co checkout
git config --global alias.sw switch
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.d diff
git config --global alias.a add
```

### Improved Log Views

```bash
# One-line log with graph
git config --global alias.lg "log --oneline --graph --decorate --all"

# Pretty formatted log with date and author
git config --global alias.lp "log --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit"

# Recent commits on current branch
git config --global alias.last "log -1 HEAD"

# All branches log
git config --global alias.ll "log --oneline --all --graph"
```

### Undo / Fix Shortcuts

```bash
# Undo last commit (keep changes staged)
git config --global alias.undo "reset --soft HEAD~1"

# Amend last commit without changing message
git config --global alias.amend "commit --amend --no-edit"

# Unstage all staged changes
git config --global alias.unstage "restore --staged"
```

### Utility Aliases

```bash
# Show list of all aliases
git config --global alias.aliases "config --get-regexp alias"

# List branches sorted by last commit date
git config --global alias.brd "branch --sort=-committerdate"

# Show what will be pushed
git config --global alias.pushwhat "log origin/main..main --oneline"

# Find a commit by message
git config --global alias.find "log --all --oneline --grep"
```


## Creating Shell Command Aliases

You can run shell commands by prefixing with `!`:

```bash
# Delete all merged branches (except main)
git config --global alias.cleanup "!git branch --merged | grep -v 'main' | xargs git branch -d"

# Create and switch to branch
git config --global alias.new "!f() { git switch -c \"$1\"; }; f"

# Push current branch to origin
git config --global alias.push-current "!git push origin $(git branch --show-current)"
```


## Viewing and Managing Aliases

```bash
# List all global aliases
git config --global --list | grep alias

# Or using the alias we created:
git aliases

# Remove an alias
git config --global --unset alias.st

# Edit all settings including aliases manually
git config --global --edit
```


## Example .gitconfig Aliases Section

```ini
[alias]
    # Basic shortcuts
    st = status
    co = checkout
    sw = switch
    br = branch
    ci = commit
    d = diff
    ds = diff --staged
    a = add
    aa = add .
    
    # Log views
    lg = log --oneline --graph --decorate --all
    last = log -1 HEAD --stat
    
    # Undo/Fix
    undo = reset --soft HEAD~1
    amend = commit --amend --no-edit
    unstage = restore --staged
    
    # Utility
    aliases = config --get-regexp alias
    cleanup = !git branch --merged | grep -v 'main' | xargs git branch -d
```

Setting up a good set of aliases can save you dozens of keystrokes every hour. Start with the aliases you use most, and build up from there.
