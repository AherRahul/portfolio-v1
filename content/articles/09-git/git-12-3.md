# Semantic Versioning with Git Tags

Semantic Versioning (SemVer) is a versioning standard that communicates the nature of changes between releases. When combined with Git tags, it provides a clear, structured history of your project's evolution.

## SemVer Format

```
MAJOR.MINOR.PATCH

Examples:
v1.0.0   - Initial stable release
v1.1.0   - Added new feature (backward compatible)
v1.1.1   - Bug fix (backward compatible)
v2.0.0   - Breaking changes (not backward compatible)
```

### When to increment each:

| Component | Increment when... |
|-----------|-------------------|
| **MAJOR** | Incompatible API changes (v1.x → v2.0.0) |
| **MINOR** | New backward-compatible features (v1.1 → v1.2.0) |
| **PATCH** | Backward-compatible bug fixes (v1.1.0 → v1.1.1) |

## Pre-Release Tags

```bash
# Alpha: very early, unstable
git tag -a v2.0.0-alpha.1 -m "Alpha: basic structure"

# Beta: feature complete, still testing
git tag -a v2.0.0-beta.1 -m "Beta: core features done"
git tag -a v2.0.0-beta.2 -m "Beta 2: bug fixes"

# Release Candidate: about to be released
git tag -a v2.0.0-rc.1 -m "Release candidate 1"

# Stable release
git tag -a v2.0.0 -m "Stable release"
```

SemVer precedence (from highest to lowest priority):
`2.0.0-alpha.1` < `2.0.0-beta.1` < `2.0.0-rc.1` < `2.0.0`

## Git Tag for Releases Workflow

```bash
# Verify main is ready
git switch main
git pull origin main
git status   # Must be clean

# Check recent history
git log --oneline -5

# Create the tag
git tag -a v1.3.0 -m "Release v1.3.0: User Dashboard

Features:
- Add user activity dashboard
- Add downloadable reports (CSV export)
- Add date range filtering

Bug Fixes:
- Fix session timeout not working on mobile
- Fix export button not visible on small screens
"

# Push the tag
git push origin v1.3.0

# Create GitHub Release from this tag
gh release create v1.3.0 --title "v1.3.0 — User Dashboard" --generate-notes
```

## Automated Version Bumping

Many projects automate version bumping using tools:

### npm / Node.js

```bash
# Automatically bumps package.json version + creates git tag
npm version patch   # 1.0.0 → 1.0.1
npm version minor   # 1.0.0 → 1.1.0
npm version major   # 1.0.0 → 2.0.0

# Then push
git push origin main --follow-tags
```

### semantic-release

A fully automated version management tool:

```bash
npm install --save-dev semantic-release

# In your CI, runs on merge to main:
# 1. Analyzes commit messages
# 2. Determines version bump (feat=minor, fix=patch, BREAKING=major)
# 3. Updates version
# 4. Creates Git tag
# 5. Generates CHANGELOG.md
# 6. Creates GitHub Release
```

## Viewing Version History

```bash
# List all version tags sorted
git tag -l "v*" --sort=version:refname

# List with dates
git for-each-ref --sort=version:refname refs/tags --format='%(refname:short) %(creatordate:short)'

# Find what changed between two versions
git log v1.0.0..v1.1.0 --oneline

# Generate changelog between versions
git shortlog v1.0.0..v1.1.0 --no-merges
```

## Build Metadata

SemVer allows build metadata after a `+`:

```
v1.0.0+build.123
v1.0.0+sha.abc1234
```

This is informational only (doesn't affect version precedence).
