# git reflog — Git's Safety Net

`git reflog` records the history of every time a branch tip or HEAD has been updated. Even when commits become "unreachable" (after deleting a branch or hard reset), reflog keeps a record, letting you recover seemingly lost work.

## What is the Reflog?

The reflog is a **local log** that records what HEAD and branch tips pointed to over time. It's like a journal of your locally performed Git operations, kept for 90 days by default.

```bash
git reflog
# or
git log -g
```

Output:

```
abc1234 (HEAD -> main) HEAD@{0}: commit: Add payment module
def5678 HEAD@{1}: commit: Add user authentication
g9h0i1j HEAD@{2}: checkout: moving from feature to main
k2l3m4n HEAD@{3}: rebase (finish): returning to refs/heads/main
p5q6r7s HEAD@{4}: reset: moving to HEAD~3
```

## Understanding the Output

Each entry shows:
- `HEAD@{0}` — Most recent position (0 = current, 1 = one step ago, etc.)
- The commit hash at that position
- What operation caused the change

## Recovering Lost Commits

### Scenario: Accidental git reset --hard

```bash
# You accidentally ran this
git reset --hard HEAD~3   # Lost 3 commits!

# Find those commits in reflog
git reflog

# HEAD@{1} was before the reset — that's what you want
abc1234 HEAD@{1}: commit: Add critical feature

# Restore to that state
git reset --hard HEAD@{1}
# or
git reset --hard abc1234
```

### Scenario: Deleted Branch with Commits

```bash
# Branch deleted before merging!
git branch -D feature/new-ui

# Find the last commit on that branch in reflog
git reflog | grep "feature/new-ui"
# or search for the commit you remember

abc1234 HEAD@{15}: checkout: moving from feature/new-ui to main

# Recreate the branch at that commit
git branch feature/new-ui abc1234
# Branch restored!
```

### Scenario: Bad Rebase

```bash
# Rebase went wrong
git rebase main  # Now my branch looks wrong

# Find ORIG_HEAD (set before rebase started)
git reflog | head -20
# Look for "rebase (start): checkout..."

# Or simply:
git reset --hard ORIG_HEAD  # Go back to before the rebase
```

## Reflog for Specific Branch

```bash
# Show reflog for a specific branch
git reflog show main

# Show HEAD reflog (default)
git reflog show HEAD
# same as:
git reflog
```

## Reflog Expiry

Reflog entries expire after 90 days (unreachable commits expire after 30 days):

```bash
# View current expiry settings
git config gc.reflogExpire
# Output: 90 days

# Change expiry
git config gc.reflogExpire "180 days"

# Never expire (not recommended)
git config gc.reflogExpire "never"

# Manually expire all reflog entries
git reflog expire --all --expire=now
```

## Viewing with Time Filter

```bash
# See where HEAD was 1 hour ago
git diff HEAD@{1.hour.ago} HEAD

# See what was committed yesterday
git log HEAD@{yesterday}..HEAD

# Show reflog entries from past week
git reflog --since="1 week ago"
```

## When Reflog Doesn't Help

Reflog is **local only** — it's not shared when you push or clone a repository:
- It won't help if someone else lost commits on their machine
- It won't help after a fresh clone (the reflog starts empty)
- After `git gc` with expired entries, old commits may be truly gone

This is why it's critical to push important work to a remote regularly.
