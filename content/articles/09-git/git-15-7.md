# git stash Advanced Techniques

The basic `git stash` and `git stash pop` covers most cases, but Git stash has many powerful features for complex scenarios.

## Named Stashes

Give your stashes descriptive names:

```bash
# Stash with a descriptive message
git stash push -m "WIP: half-finished payment refactor"
git stash push -m "Experimental: trying different API structure"

# List shows your names
git stash list
# stash@{0}: On main: Experimental: trying different API structure
# stash@{1}: On main: WIP: half-finished payment refactor
# stash@{2}: WIP on main: abc1234 Normal commit message
```

## Stashing Specific Files

```bash
# Stash only specific files
git stash push src/auth.js src/payment.js -m "Auth and payment WIP"

# Stash all untracked files too (new files)
git stash push --include-untracked
git stash push -u   # Short form

# Stash everything including ignored files
git stash push --all
git stash push -a   # Careful: this includes node_modules, etc.!

# Interactive stash — choose which hunks to stash
git stash push --patch
git stash push -p   # Like git add -p, but for stashing
```

## Applying Specific Stashes

```bash
# List all stashes
git stash list

# Apply a specific stash (by index)
git stash apply stash@{2}

# Pop a specific stash
git stash pop stash@{1}

# Preview a stash before applying
git stash show stash@{0}           # Summary of changes
git stash show -p stash@{0}        # Full diff
```

## Stash Branching

If applying a stash might cause conflicts, create a branch from the stash:

```bash
# Create a branch, checkout the commit where stash was made, apply stash
git stash branch new-feature stash@{0}
# This creates: git checkout -b new-feature <stash-parent-commit>
# Then applies the stash
# Then drops the stash if it applied cleanly
```

This is safer than trying to apply a stash on a diverged branch.

## Dropping Stashes

```bash
# Drop the top stash
git stash drop

# Drop a specific stash
git stash drop stash@{2}

# Drop all stashes
git stash clear
```

## Viewing Stash Contents

```bash
# Show stats of the most recent stash
git stash show

# Show full diff of most recent stash
git stash show -p

# Show stats of a specific stash
git stash show stash@{1}

# Show stash diff with --stat
git stash show --stat stash@{0}
```

## Stash with Untracked Files

By default, `git stash` only stashes tracked modified files. For new files:

```bash
# Create some new untracked files
touch new-feature.js new-test.js

git stash          # Does NOT stash these new files!
git status         # Shows new-feature.js and new-test.js still there

git stash -u       # NOW stashes untracked files too
git status         # Clean!
```

## Stash on a Different Branch

You stashed on wrong branch? Apply it on the right one:

```bash
git switch feature/correct-branch
git stash apply    # Apply stash from wherever you made it
```

The stash is independent of branches — you can apply any stash to any branch.

## Recovering Lost Stashes

If you accidentally dropped a stash:

```bash
# Find dangling stash commits
git fsck --unreachable | grep commit

# For each SHA shown, inspect it
git show <sha>

# If it's your stash, restore it
git stash apply <sha>
```

Unlike committed code, dropped stashes may not always be recoverable — they'll eventually be garbage collected.
