# Git LFS — Large File Storage

Git is optimized for text files. Large binary files (images, videos, compiled binaries, datasets) make repositories slow and bloated. **Git Large File Storage (LFS)** is an extension that stores large files externally while keeping small pointer files in the repository.

## How Git LFS Works

Without LFS:
```
Repository stores full binary file directly → Large repo size
Every git clone downloads all versions of every binary file
```

With LFS:
```
Repository stores: tiny pointer file (a few bytes)
LFS server stores: the actual binary file

When you checkout:
  → Git reads pointer → Fetches actual file from LFS server
```

## Installation

```bash
# Install Git LFS
# macOS:
brew install git-lfs

# Ubuntu/Debian:
sudo apt install git-lfs

# Windows (via Chocolatey):
choco install git-lfs

# Initialize LFS in Git
git lfs install   # Run once after installation
```

## Setting Up LFS in a Repository

```bash
# Navigate to your repository
cd my-project

# Track specific file types with LFS
git lfs track "*.psd"          # Photoshop files
git lfs track "*.png"          # PNG images
git lfs track "*.mp4"          # Videos
git lfs track "*.zip"          # Archives
git lfs track "*.pdf"          # PDFs
git lfs track "models/*.bin"   # ML models

# This creates/updates .gitattributes
cat .gitattributes
# *.psd filter=lfs diff=lfs merge=lfs -text
# *.png filter=lfs diff=lfs merge=lfs -text

# Commit .gitattributes
git add .gitattributes
git commit -m "Configure Git LFS tracking"
```

## Tracking Files

```bash
# Track by pattern
git lfs track "*.jpg"

# Track a specific file
git lfs track "data/dataset.csv"

# List all tracked patterns
git lfs track

# Or view .gitattributes
cat .gitattributes
```

## Adding, Committing, and Pushing LFS Files

Once tracking is configured, Git LFS is transparent — use normal Git commands:

```bash
# Add a tracked file (LFS handles it automatically)
git add images/hero.png

# Commit
git commit -m "Add hero image"

# Push (LFS files are uploaded separately)
git push origin main
# Uploading LFS objects: 100% (1/1), 4.5 MB | 1.0 MB/s, done.
```

## Cloning LFS Repositories

```bash
# Normal clone - also downloads LFS files
git clone https://github.com/user/repo.git

# Skip LFS downloads (get pointer files only - saves bandwidth)
GIT_LFS_SKIP_SMUDGE=1 git clone https://github.com/user/repo.git

# Download LFS files for specific patterns later
git lfs pull --include="*.png"

# Download all LFS files
git lfs pull
```

## Migrating Existing Large Files to LFS

If you already have large files in history:

```bash
# Migrate tracked types to LFS (rewrites history!)
git lfs migrate import --include="*.psd,*.png,*.mp4"

# Then force push
git push --force-with-lease origin --all
```

## Useful LFS Commands

```bash
# See LFS status
git lfs status

# See tracked LFS files in working directory
git lfs ls-files

# See LFS environment info
git lfs env

# Fetch LFS objects without checkout
git lfs fetch origin main

# Prune local LFS cache (old/unused files)
git lfs prune
```

## GitHub LFS Pricing

GitHub provides 1GB free storage and 1GB/month bandwidth for LFS. Additional storage/bandwidth requires a paid plan.

For large projects, consider:
- **DVC (Data Version Control)** for large ML datasets
- **GitHub packages** for compiled binaries
- **Cloud storage** (S3, GCS) for very large assets
