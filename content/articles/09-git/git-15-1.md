# git stash

`git stash` temporarily shelves (or "stashes") changes you've made to your working directory so you can switch to a different task, without committing incomplete work. Think of it as a "clipboard" for uncommitted work.


## When to Use git stash

- You're in the middle of a feature and suddenly need to fix an urgent bug on another branch.
- You want to switch branches but have uncommitted changes that would get in the way.
- You want to pull the latest changes without committing your current WIP.
- You made changes on the wrong branch and want to move them to the correct branch.


## Basic Stash Operations

### Stashing Changes

```bash
# Stash all tracked changes (staged and unstaged)
git stash

# Stash with a descriptive message
git stash save "WIP: implementing payment gateway"
# (modern syntax:)
git stash push -m "WIP: implementing payment gateway"

# Stash including untracked files
git stash --include-untracked
# or shorter:
git stash -u

# Stash everything including ignored files
git stash --all
```

### Listing Stashes

```bash
git stash list
```

Output:

```
stash@{0}: On feature/login: WIP: implementing payment gateway
stash@{1}: On main: Fix in progress
stash@{2}: WIP on feature/search: Add search filters
```

Stashes are numbered with `{0}` being the most recent.

### Applying a Stash

```bash
# Apply most recent stash (and keep it in the stash list)
git stash apply

# Apply a specific stash
git stash apply stash@{2}

# Apply and remove from stash list (pop)
git stash pop

# Pop a specific stash
git stash pop stash@{1}
```

**Difference between `apply` and `pop`:**
- `apply`: Applies the stash but keeps it in the stash list
- `pop`: Applies the stash and removes it from the stash list


### Removing Stashes

```bash
# Remove a specific stash
git stash drop stash@{1}

# Remove all stashes
git stash clear
```


## Advanced Stash Operations

### Stash Only Specific Files

```bash
# Stage specific files, then stash only staged changes
git add src/auth.js
git stash --staged
# (only src/auth.js is stashed, other changes remain)

# Or use patch mode for interactive selection
git stash -p   # Interactively select which changes to stash
```

### Viewing Stash Contents

```bash
# Show a summary of the most recent stash
git stash show

# Show the full diff of the most recent stash
git stash show -p

# Show a specific stash
git stash show -p stash@{2}
```

### Creating a Branch from a Stash

When you want to apply stashed changes to a new branch:

```bash
git stash branch feature/incomplete-work stash@{0}
# Creates a new branch and applies the stash to it
```


## Typical Workflow

```bash
# 1. You're mid-feature and need to switch to a hotfix
git stash push -m "WIP: user dashboard"

# 2. Fix the bug on main
git switch main
git pull
git switch -c hotfix/login-null-pointer
# ... fix and commit ...

# 3. Return to your feature
git switch feature/dashboard
git stash pop

# 4. Continue where you left off
```


## Handling Stash Conflicts

When applying a stash, conflicts can occur (just like merges):

```bash
git stash pop
# CONFLICT in src/app.js
```

Resolution:

```bash
# Fix conflicts
git add src/app.js
# Complete the stash application (no commit needed for stash apply)
git restore --staged src/app.js  # If you accidentally staged during conflict

# Note: After a conflict during 'pop', the stash is NOT automatically dropped
# Check it's gone or drop it manually:
git stash list   # Check
git stash drop   # Drop if still there
```


## Summary

```bash
git stash              # Save all changes
git stash -u           # Save including untracked files
git stash push -m "msg"  # Save with description
git stash list         # List all stashes
git stash pop          # Apply and remove most recent stash
git stash apply stash@{N}  # Apply specific stash (keep it)
git stash drop stash@{N}   # Delete specific stash
git stash clear        # Delete all stashes
git stash show -p      # View stash diff
```
