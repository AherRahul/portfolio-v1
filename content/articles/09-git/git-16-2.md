# Advanced Git Configuration

Beyond basic `git config user.name`, Git has many powerful configuration options for performance, security, credential management, and workflow customization.

## Configuration Levels

```bash
# System-wide (all users): /etc/gitconfig
git config --system core.editor vim

# User-wide (current user): ~/.gitconfig
git config --global core.editor code

# Repository-specific: .git/config
git config core.autocrlf false

# Highest priority wins: repository > user > system
```

## Viewing Configuration

```bash
# See all config with source file
git config --list --show-origin

# See effective config
git config --list

# See a specific setting
git config core.editor
```

## Conditional Includes (Per-Directory Config)

Use different identities for work vs. personal projects:

```ini
# ~/.gitconfig
[user]
    name = Jane Doe
    email = jane@personal.com

[includeIf "gitdir:~/work/"]
    path = ~/.gitconfig-work

[includeIf "gitdir:~/opensource/"]
    path = ~/.gitconfig-oss
```

```ini
# ~/.gitconfig-work
[user]
    name = Jane Doe
    email = jane@company.com
    signingkey = WORK_GPG_KEY

[commit]
    gpgsign = true
```

Now any repository under `~/work/` uses the work email automatically!

## Performance Settings

```bash
# Speed up status on large repositories
git config core.untrackedCache true
git config feature.manyFiles true

# Parallel fetch
git config submodule.fetchJobs 4

# Filesystem monitor (watch file changes in background)
git config core.fsmonitor true   # macOS/Linux with watchman

# Pack compression level (faster with less compression)
git config pack.compression 1

# Large files - increase limits
git config pack.packSizeLimit 2g
```

## Credential Storage

```bash
# Cache credentials in memory for 1 hour
git config --global credential.helper 'cache --timeout=3600'

# Store credentials on disk (plain text - careful!)
git config --global credential.helper store

# Use OS keychain (macOS)
git config --global credential.helper osxkeychain

# Use Windows Credential Manager
git config --global credential.helper manager-core

# Use Git Credential Manager (cross-platform)
git config --global credential.helper manager
```

## Useful Global Settings

```bash
# Set default branch name for new repositories
git config --global init.defaultBranch main

# Automatically prune on fetch
git config --global fetch.prune true

# Default rebase instead of merge on git pull
git config --global pull.rebase true

# Show rebase instructions in a more readable format
git config --global rebase.abbreviateCommands true

# Set merge tool
git config --global merge.tool vscode
git config --global mergetool.vscode.cmd 'code --wait $MERGED'

# Diff tool
git config --global diff.tool vscode
git config --global difftool.vscode.cmd 'code --wait --diff $LOCAL $REMOTE'

# GPG signing
git config --global commit.gpgsign true
git config --global user.signingkey YOUR_GPG_KEY_ID

# Auto color
git config --global color.ui auto

# Show the origin of a ref in log
git config --global log.decorate auto

# Push current branch by default
git config --global push.default current

# Reuse Recorded Resolution
git config --global rerere.enabled true

# Verbose push output
git config --global push.followTags true
```

## Per-URL Credential Config

```bash
# Use different credentials for different GitHub accounts
git config --global credential.https://github.com.username personaluser
git config --global credential.https://work.github.com.username workuser
```

## Aliases (Advanced)

```ini
# ~/.gitconfig
[alias]
    lg = log --oneline --graph --all --decorate
    undo = reset --soft HEAD~1
    amend = commit --amend --no-edit
    aliases = config --get-regexp alias
    root = rev-parse --show-toplevel
    ignored = ls-files --others -i --exclude-standard
    tracked = ls-files --others --exclude-standard
    new = !f() { git switch -c "$1"; }; f
    save = !git add -A && git commit -m 'SAVEPOINT'
    wip = commit -am "WIP"
    unwip = reset HEAD~1 --mixed
```

## Editing .gitconfig Directly

```bash
git config --global --edit
```

Opens the full `~/.gitconfig` in your default editor for direct editing.
